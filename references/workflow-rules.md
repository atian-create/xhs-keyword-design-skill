# Keyword Placement Rules

Use these rules when running `$xhs-keyword-design`.

## Inputs To Check

- title
- cover copy
- first 3 lines or first 3 seconds
- body copy, script, or page text
- intended audience and track
- existing keyword list, if any

## Workflow Rules

- Identify the core search word.
- Add industry and category words.
- Add audience or life-stage words.
- Add pain, result, and scenario words.
- Limit the final keyword field to 10.
- Check whether keywords appear naturally in title, cover, opening, body, and keyword field.

## Output Must Include

- final 10 keywords
- keyword type labels
- where each keyword should appear
- natural embedding suggestions
- warning if keywords do not match the content
- optional revised title or opening keyword placement

## Quality Checks

- The output must be usable from the material provided by the user.
- Recommendations must explain their reasoning.
- If evidence is weak or missing, mark it clearly.
- Keep the first result focused; do not expand into a full consulting report by default.
- Do not claim automatic publishing, guaranteed growth, guaranteed traffic, or guaranteed sales.
- Do not copy another creator's exact wording, images, story, identity, or claims.

## Public-Safe Wording

Prefer:

- workflow
- checklist
- review
- judgment
- public signal
- user-provided material
- next action

Avoid:

- private data extraction
- secret system
- guaranteed viral result
- guaranteed traffic growth
- copy-paste another creator's content
